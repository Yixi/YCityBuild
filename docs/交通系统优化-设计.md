# 交通系统优化设计 — 车道 · 跟车 · 信号灯

> **定位**：把当前「代表性车辆」从「沿路线中心点滑动、会重叠成团」升级为**真实的微观交通可视化**：双向车道、跟车排队、交叉口信号灯与让行、平滑转弯。
> **前置**：阶段 2 已实现统计式通勤/货运流（`sim/traffic.ts` 按 `world.routes` 聚合路段拥堵）与车辆池（`agents/vehicles.ts`）。本设计在其上重写车辆层。

---

## 一、现状问题（对照截图）

当前 `agents/vehicles.ts` 的局限：

| 问题 | 代码原因 |
|---|---|
| **车辆重叠成团** | 位置取路段中心 `_p.set(px, 0.12, pz)`，无车道偏移；同一格多车完全重合 |
| **无双向车道** | 不区分行驶方向，去/回程车挤在同一条中线 |
| **互相穿透、不排队** | 速度只看聚合拥堵 `world.traffic[cur]`，无「前车间距」概念 |
| **拐角穿楼/急转** | 直接在格中心间线性插值，交叉口不转弯 |
| **无交叉口规则** | 没有停止线、信号灯、让行；车流在路口直接交叠 |
| **聚团** | 随机抽路线 + 随机起点，大量车落在同几条主路 |

核心：车辆是「沿折线滑动的贴图」，不是「有车道、有间距、守信号的 agent」。

---

## 二、目标与关键决策

**目标**：忠实的微观交通可视化——**双向车道分离 + 跟车排队 + 交叉口信号灯/让行 + 平滑转弯**，封顶数百辆仍流畅。

**关键架构决策（建议）**：
- **经济用的拥堵保持「统计式」**（`sim/traffic.ts` 按路线量聚合，确定性、驱动通勤成本/经济）——**不动**，保证可复现与 golden-test。
- **车辆层做「忠实的视觉微观模拟」**（车道/跟车/信号），用 `Math.random` 等视觉抖动即可，**不回写逻辑**。
- 可选增强：把车辆**实际排队长度**回写为一层「视觉拥堵」用于热力图润色，但经济仍用统计值。

> 这样既得到「看得见的真实交通」，又不破坏模拟确定性与现有经济平衡。

## 三、详细设计

### 1. 车道模型与道路几何（`agents/laneGraph.ts`，新增）
- **有向车道**：每条道路格在每个行驶方向上有 1 条车道（双向共 2 条；未来大道可扩为多车道）。
- **车道偏移**：车辆从格 A→B，方向 `d`，向 `d` 的**右手侧**偏移 `LANE_OFFSET≈0.22`（右行制），去/回程自然分离 → 根除重叠。
- **车道折线 + 转弯**：把路线 `path=[cell…]` 转成**车道点折线**：直行段取偏移后的格中心点；在交叉口对相邻三元组 `(prev,cur,next)` 插入**圆角/贝塞尔转弯点**，让车在路口内沿弧线转向，不再切角穿楼。
- **车道图**：以「有向边 = 相邻两道路格 + 方向」为节点，预计算每条边的入/出车道接点与转弯曲线；路线复用缓存（仅 `routesDirty` 时重建）。

### 2. 车辆 agent：车道跟随 + 跟车模型（重写 `agents/vehicles.ts`）
- 车辆状态：`{ lanePath: Vec2[], s: 弧长进度, speed, ... }`，沿**车道折线**按弧长前进（而非格中心线性插值）。
- **跟车（IDM-lite / 间距法）**：每帧查询**同车道前方最近车**的间距 `gap`：
  - `gap < 安全距`：减速/停车（`speed → 0`）；
  - 否则向自由流速度 `freeSpeed`（受该格统计拥堵轻微调制）加速。
  - 产生真实的**排队、走走停停**，杜绝重叠/穿透。
- **解聚团**：初始与重分配时按车道**错相位**铺开（沿弧长均匀撒点），避免同点起步。

### 3. 交叉口与信号控制（`agents/trafficSignals.ts`，新增）
- **交叉口识别**：复用 `world.roadShape`——`CROSSROAD`（十字）与 `T_INTERSECTION`（丁字）即路口。
- **信号灯（十字）**：每个十字一个 `SignalController`，按相位循环：`南北绿 ↔ 东西绿`（含黄灯过渡、可配时长）。车辆在**停止线**（进入路口前的格边界）遇红灯则停（跟车模型把它当成「前方静止障碍」即可统一处理）。
- **丁字/无灯路口**：次要路让行（`stop`/`yield`）；或同样给信号灯，简化冲突。
- **路口冲突避免**：信号在时间上分离南北/东西流，停止线 + 信号即可避免直行对撞；左转简化为**绿灯期让直行**（或先只放行直行+右转，左转排队）。可加轻量「路口占用锁」防绿灯尾车滞留对撞。
- 信号状态每帧由控制器推进（与渲染解耦；可选与模拟 tick 对齐保证确定性）。

### 4. 信号灯渲染
- 程序化**信号灯杆 + 三色灯头**原型（盒体，thin-instance），置于每个路口角；当前相位对应灯头亮色（红/黄/绿，用 emissive 或换原型）。
- 备选：CC0 交通灯模型（poly.pizza）按需接入。

### 5. 空间索引（邻车查询，`agents/vehicles.ts` 内）
- 每帧重建 `Map<有向边, 按进度排序的车辆列表>`（240 车，重建成本可忽略）→ O(1) 找「前车」。
- 或用网格空间哈希 `Map<cell, Veh[]>` + 同向过滤。

### 6. 车道图寻路（路线 → 车道序列）
- `world.routes` 的 `path`（格序列）映射为**车道折线序列**（含转弯曲线）。货车/通勤共用同一车道图，仅模型与车道选择不同。
- 缓存：路网拓扑不变（`!routesDirty`）时车道折线复用。

### 7. 与统计拥堵的耦合
- **经济侧不变**：`sim/traffic.ts` 仍按 `world.routes` 聚合拥堵 → 通勤成本/经济（确定性）。
- **可选**：车辆 freeSpeed 读该格统计拥堵做轻微调制（视觉与统计一致）；并可把**实际队列密度**写入一层 `visualTraffic` 仅供热力图润色（不参与经济）。

### 8. 性能
- 车辆封顶（现 `MAX_VEHICLES=240`）；空间哈希邻车查询 O(1)。
- **LOD**：远处车辆简化/隐藏（按相机距离），只保留近景微观行为。
- 车 + 信号灯均 thin-instance；每帧仅更新可见车矩阵。
- 微观模拟 240 车/帧 + 信号，开销很小，目标维持当前 120 fps。

## 四、数据模型与模块改动

```
新增  agents/laneGraph.ts      车道几何/转弯曲线/车道图（由 world.road + roadShape 生成，路网变更时重建）
新增  agents/trafficSignals.ts 交叉口识别 + 信号相位控制器
重写  agents/vehicles.ts       车道跟随 + 跟车(IDM-lite) + 守信号 + 空间索引
新增  render 信号灯原型         程序化灯杆/三色灯头（或 CC0 模型）
微调  render/overlay.ts(可选)   新增「队列密度」视觉拥堵图层
不动  sim/traffic.ts            经济用统计拥堵保持确定性
复用  core/roadShape.ts         CROSSROAD/T_INTERSECTION 识别路口
复用  world.routes              作为车道寻路输入
```

新增常量（`core/constants.ts`）：`LANE_OFFSET`、`CAR_LENGTH`、`SAFE_GAP`、`SIGNAL_GREEN_TICKS`、`SIGNAL_YELLOW_TICKS` 等。

## 五、分阶段路线图（每阶段可独立运行验证）

| 阶段 | 内容 | 价值 |
|---|---|---|
| **P1 车道 + 偏移 + 平滑转弯** | 双向车道偏移、车道折线、路口圆角转弯 | **立刻消除重叠**，呈现双向车流、正确转弯（最大视觉收益、低风险） |
| **P2 跟车排队** | 同车道前车间距 → 减速/停车，错相位铺开 | 车辆保持车距、排队、走停，杜绝穿透/聚团 |
| **P3 交叉口信号灯** | 十字信号相位 + 停止线 + 灯柱渲染，红灯停 | 真实「等红灯」、路口不再交叠 |
| **P4 让行与冲突** | 丁字让行/停车让行、路口占用、左转让直行 | 路口秩序与安全 |
| **P5 耦合与打磨** | 队列密度→视觉拥堵图层、LOD、信号配时调优 | 性能与观感收尾 |

**建议先做 P1+P2**（解决你截图里的核心问题：重叠 + 无车道），再做 P3 信号灯。

## 六、验证

- 每阶段 `pnpm dev` 实机 + DevTools 截图：P1 看双向分车道、路口转弯不切角；P2 看排队走停、无重叠；P3 看红灯停车、信号循环。
- 帧率维持（DevTools 看 fps 与 draw call；车/灯 thin-instance）。
- 经济不受影响（统计拥堵不变，存档/确定性回归通过）。
- `pnpm typecheck/lint/build` 全绿。

## 七、风险与取舍

| 风险 | 取舍 |
|---|---|
| 全微观模拟过重 | 车辆封顶 + 空间哈希 + LOD；经济仍用统计拥堵 |
| 路口左转冲突复杂 | 先「直行+右转」放行、左转让直行；不做复杂相位优化 |
| 车道图随路网频繁重建 | 仅 `routesDirty`/路网变更时重建并缓存 |
| 确定性 | 经济统计拥堵保持确定；车辆为视觉层（可用随机），不回写逻辑 |
| 大地图车多 | LOD 隐藏远车，只在视野内做微观行为 |
